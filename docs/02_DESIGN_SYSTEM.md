# Design System

Version: 1.0

---

# Purpose

This document defines the visual language of Tradings.

Every screen, component, and interaction must follow these rules.

The goal is consistency, readability, and long-session usability.

This document has higher priority than personal preferences or AI-generated design ideas.

---

# Design Philosophy

Tradings is designed as a professional financial workspace.

The interface should maximize productivity while minimizing cognitive load.

Every visual element must have a purpose.

The design must feel calm instead of exciting.

Professional instead of decorative.

Elegant instead of flashy.

---

# Core Principles

## Information First

Data is more important than decoration.

Users open Tradings to make financial decisions.

Every pixel should support that goal.

---

## Consistency

Spacing must be predictable.

Typography must be predictable.

Cards must look related.

Buttons must behave consistently.

Never create isolated design patterns.

---

## Simplicity

Reduce visual noise.

Avoid unnecessary borders.

Avoid unnecessary shadows.

Avoid unnecessary colors.

Avoid unnecessary animations.

---

## Density

The UI should display a large amount of useful information without feeling crowded.

Whitespace should separate groups of information.

Do not waste space.

---

# Visual Inspiration

The design language may take inspiration from:

- TradingView
- Bloomberg Terminal (modern interpretation)
- Linear
- Stripe Dashboard
- Vercel Dashboard
- Apple Human Interface Guidelines

Never clone another product.

Create an original identity.

# Shadows

Keep shadows minimal.

Most cards should rely on contrast instead of shadows.

Avoid floating card appearance.

---

# Borders

5%

Avoid heavy outlines.

# Layout Rhythm

Components should align to a consistent spacing rhythm.

Use whitespace to separate sections.

Avoid excessive empty areas.

Avoid cramped layouts.

---

# Cards

Cards represent groups of related information.

Every card must:

- Have consistent padding.
- Have consistent radius.
- Use subtle borders.
- Avoid heavy shadows.
- Avoid decorative gradients.

---

# Buttons

Buttons should communicate hierarchy.

Primary Button

Blue background

Secondary Button

Surface background

Danger Button

Red background

Ghost Button

Transparent

Never invent new button styles.

---

# Inputs

Inputs should feel lightweight.

Border only.

No heavy shadows.

Rounded 8px.

---

# Tables

Tables should prioritize readability.

Compact row height.

Consistent alignment.

Right-align numbers where appropriate.

---

# Charts

Charts are the visual focus.

The chart should always occupy the largest visual area.

Never reduce chart size to make room for decorative cards.

---

# Responsive Design

Desktop First

Tablet Second

Mobile Third

Never compromise desktop usability.

---

# Animation

Animations must support understanding.

Never distract.

Recommended duration:

120ms

150ms

200ms

Avoid long animations.

---

# Accessibility

Maintain sufficient contrast.

Keyboard navigation must work.

Focus indicators must remain visible.

Never rely on color alone.

---

# Definition

Whenever a design decision is unclear:

Choose the option that improves readability, consistency, and productivity.
