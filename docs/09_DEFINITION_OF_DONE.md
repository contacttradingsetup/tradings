# Definition of Done

Version: 1.0

---

A task is NOT finished until every item below is completed.

---

# Functionality

The feature works correctly.

No known bugs.

No broken navigation.

---

# UI

Matches the Design System.

Consistent spacing.

Consistent typography.

Consistent colors.

Responsive.

Accessible.

No visual glitches.

---

# Components

Reusable.

No duplicated components.

No unnecessary props.

Typed correctly.

---

# Code

TypeScript passes.

ESLint passes.

No unused imports.

No dead code.

No unnecessary comments.

Readable.

Maintainable.

---

# Performance

No unnecessary re-renders.

Optimized images.

Lazy loading where appropriate.

No obvious performance regressions.

---

# Security

Input validated.

Sensitive data protected.

No secrets committed.

---

# Documentation

Relevant documentation updated.

Architecture still valid.

Design System still valid.

---

# Testing

Existing tests still pass.

New critical logic tested (when applicable).

---

# Review Checklist

Before merging, verify:

- No duplicated code
- No duplicated Tailwind classes
- No random spacing values
- No inconsistent typography
- No unnecessary border radius
- No unnecessary animations
- No hardcoded colors outside the design system
- Components follow naming conventions
- Imports are clean
- Folder structure remains organized

---

# Final Question

Can another engineer understand and maintain this code six months from now?

If the answer is "No",

the task is not done.
